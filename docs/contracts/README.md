# Contracts

`docs/contracts/` contains normative file and API contracts.

## Active State

- `CURRENT.md` names the active major version.
- `v1/` contains the active post code data contract and the static API contract.

## Rules

- Contract changes must be covered by tests or repository checks.
- Breaking changes require a new major version or a dedicated ADR.
- Ops and plan documents must not redefine contracts.
