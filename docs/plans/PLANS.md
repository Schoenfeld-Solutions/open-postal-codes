# ExecPlan Standard for Open Postal Codes

This file is the canonical rule base for new plan documents in the repository.

## Purpose

Plans should be written so Gabriel or a new technical contributor can implement the change without context from other repositories.

## Scope

- New plan documents live under `docs/plans/`.
- Active plans follow `docs/plans/PLANS-<number>-<slug>.md`.
- Plans consider only this repository.

## Required Structure

Active plans contain:

- `## Purpose / Big Picture`
- `## Progress`
- `## Surprises & Discoveries`
- `## Decision Log`
- `## Outcomes & Retrospective`
- `## Context and Orientation`

The `Progress` section contains at least one checkbox entry.
